# Code of Conduct

Be direct, respectful, and specific.

This project welcomes contributors who improve the skills library, CLI, documentation, and examples. Keep feedback about the work, avoid personal attacks, and assume maintainers are optimizing for a useful public resource.

Unacceptable behavior includes harassment, threats, doxxing, spam, and repeated disruptive comments after a maintainer has asked for a change in approach.

Maintainers may hide comments, close issues, or block participants when needed to keep the project usable.
